## Installation / Porting to Another Engine Build

### Quick Start

Copy the entire `Engine/Plugins/UnrealEngineAIAssist/` directory to the target engine,
then run the setup script:

```bash
cd Engine/Plugins/UnrealEngineAIAssist
python setup.py --client claude    # or: codebuddy, cursor, windsurf
python setup.py --check            # verify installation
```

See `README.md` for detailed instructions.

### Manual Installation

#### Step 1: Copy the Plugin

Copy `Engine/Plugins/UnrealEngineAIAssist/` to the target engine's `Plugins/` directory.

Requirements:
- UE 4.26+ (uses `UProperty`, `TFieldIterator`, `TPromise/TFuture`)
- 6 module dependencies only: `Core`, `CoreUObject`, `Sockets`, `Networking`, `Json`, `JsonUtilities`
- PythonScriptPlugin is a **soft dependency** (headers resolved via `PrivateIncludePaths` + `#if __has_include`). The plugin compiles and runs without Python — `exec_python` will return an error, but `describe_object` and `generate_catalog` work fine.

Then regenerate project files and rebuild the editor:
```
# Windows
GenerateProjectFiles.bat
# Then build via Visual Studio
```

On startup, verify the log shows:
```
UnrealEngineAIAssist: Listening on 127.0.0.1:13090
```

Custom port: launch with `-AgentBridgePort=14000`.

#### Step 2: Run setup.py (or manually configure)

The setup script handles skill symlinks, MCP config, and Python venv:
```bash
python Engine/Plugins/UnrealEngineAIAssist/setup.py --client claude
```

Or manually:
1. Create `Engine/.{agent}/skills/` with symlinks to each skill in the plugin's `Skills/` directory
2. Create/merge `Engine/.mcp.json` pointing to the MCP server
3. Install Python deps: `cd Skills/ue-python-script/mcp_server && uv sync`

#### Step 3: Generate the Catalog

With the editor running, call `generate_catalog()` via MCP (or ask the AI to do it).
This populates `.{agent}/callable_catalog/` with the JSON files.

### Verification Checklist

1. Editor log: `UnrealEngineAIAssist: Listening on 127.0.0.1:13090`
2. TCP test: `{"command":"ping","params":{}}\n` → `{"success":true,"message":"pong"}\n`
3. MCP tools: Claude Code shows `exec_python`, `describe_object`, `generate_catalog`, `reflect` in available tools
4. Catalog: `catalog_index.json` exists with 50+ classes after generation

## MCP Tools

| Tool | Returns Inline | Dumps to File |
|------|---------------|---------------|
| `exec_python(code)` | Full result + log_output | — |
| `describe_object(class_name)` | Summary (class, parent, counts, preview) | Full JSON → `mcp_output/describe_object_{class}.json` |
| `generate_catalog(output_dir?)` | Summary (counts, path) | Full catalog → `callable_catalog/` |
| `get_log(count?, category?, verbosity?, filter?)` | Summary (counts, recent errors/warnings) | Full log → `mcp_output/log_{filter}_{ts}.txt` |
| `reflect(action, object?, property?, value?, class_name?)` | get/set: full result; describe: summary | describe → `mcp_output/reflect_describe_{class}.json` |

### Context-Saving Strategy

`describe_object` and `get_log` return **compact summaries** inline (~200 tokens) and dump
full data to `.{agent}/mcp_output/`. Use `Read` or `Grep` on the `file_path`
to find specific information without loading everything into context.

**Example:** After `describe_object("Actor")` returns `file_path`, search for a specific function:
```
Grep(pattern="get_all_actors", path=file_path)
```

## 3-Level Discovery Workflow

### Level 0: Category Index
Read `.{agent}/callable_catalog/catalog_index.json` to find relevant categories:
- `actor_management`, `asset_management`, `level_management`, `rendering`,
  `blueprint_editing`, `mesh_geometry`, `animation`, `physics`, `ui_umg`, `general`

Each category lists its classes and function count.

### Level 1: Class Details
Read `.{agent}/callable_catalog/classes/{ClassName}.json` for full function signatures:
- `name`, `python_name`, `is_static`, `is_pure`, `safety_level`
- `params` with types, `return_type`, `python_snippet`

### Level 2: Live Introspection
Call `describe_object("ClassName")` for real-time detail:
- All functions including inherited ones (Level 1 only has class-own functions)
- All BlueprintVisible properties with types
- Parameter details with python types

**Example flow:** User asks "how do I get all static mesh actors?"
1. Read catalog_index.json -> category `actor_management` -> class `GameplayStatics`
2. Read classes/GameplayStatics.json -> function `get_all_actors_of_class`
3. Compose and execute:
```python
import unreal
actors = unreal.GameplayStatics.get_all_actors_of_class(
    unreal.EditorLevelLibrary.get_editor_world(),
    unreal.StaticMeshActor
)
print(f"Found {len(actors)} static mesh actors")
```

## Raw Reflection Access (`reflect`)

The `reflect` tool bypasses UE's property permission gates (`CPF_BlueprintVisible`, `CPF_BlueprintReadOnly`, `CPF_EditConst`, etc.) using `FProperty::ImportText()` / `ExportTextItem()`. This unlocks ~30-40% of engine properties that are inaccessible via Python's `set_editor_property`.

### When to Use

- Property is not `BlueprintVisible` (e.g., `WidgetTree.RootWidget` — `UPROPERTY(Instanced)`)
- Property is `BlueprintReadOnly` but you need to write (e.g., `DataTable.RowStruct`)
- Property has `EditConst` or `DisableEditOnInstance/Template` flags
- You need to inspect internal/private UPROPERTY fields for debugging

### Actions

#### `reflect describe` — Discover all properties (no filter)
```
reflect(action="describe", class_name="WidgetTree")
→ Lists ALL properties with accessible_via: "python" | "python_read_only+reflect_write" | "reflect"
→ Lists ALL functions with accessible_via: "python" | "python_call_method"
→ Dumps full data to file, returns summary
```

#### `reflect get` — Read any property
```
reflect(action="get", object="/Game/UI/WBP.WBP:WidgetTree", property="RootWidget")
→ {"success": true, "value": "CanvasPanel_0", "type": "UWidget*"}
```

#### `reflect set` — Write any property (bypasses all permission gates)
```
reflect(action="set", object="/Game/UI/WBP.WBP:WidgetTree", property="RootWidget",
        value="/Game/UI/WBP.WBP:CanvasPanel_0")
→ {"success": true, "previous_value": "None", "new_value": "...CanvasPanel_0",
   "warning": "UNSAFE: Bypassed property permission gates via ImportText."}
```

### Value Format (Unreal Text Serialization)
- Primitives: `"42"`, `"3.14"`, `"true"`, `"Hello World"`
- Object references: full object path string
- Structs: `"(X=1.0,Y=2.0,Z=3.0)"`
- Enums: `"EnumValue"`

### Dot-Path Property Chains
For nested properties, use dot-separated paths: `"Outer.Inner.Leaf"`.
Each intermediate segment must be an object property.

### Discovery Workflow
1. `reflect(action="describe", class_name="MyClass")` — see which properties need reflect
2. Use `set_editor_property` for `accessible_via: "python"` properties (safer, with validation)
3. Use `reflect get/set` only for `accessible_via: "reflect"` or `"python_read_only+reflect_write"` properties

### ⚠️ Safety
- `reflect set` bypasses ALL permission checks — no type safety, no editor validation
- Always returns `previous_value` so changes can be manually reverted
- Always fires `PostEditChangeProperty` to notify the editor
- Prefer `set_editor_property` when the property is Python-accessible

## Safety Protocol

Check the `safety_level` field in the catalog before executing:

| Level | Action |
|-------|--------|
| `read_only` | Execute immediately (Get, Find, Is, Has, Count, Check, Query) |
| `editor_modify` | Inform the user what will change, then execute. Remind about Ctrl+Z. |
| `destructive` | **Require explicit user confirmation** (Delete, Destroy, Remove, Clear, Reset, Purge) |

**Escalation rules:**
- When in doubt, treat as the higher risk level
- Batch operations on many objects always require confirmation, even for `editor_modify`
- Never retry destructive operations automatically on failure

## When the Editor is Not Running

If MCP connection fails (`ConnectionRefusedError`):
- **Catalog search** still works (read local JSON files)
- **Code composition** still works (build snippets from catalog data)
- **Execution** is unavailable — inform the user the editor must be running
- For catalog generation without editor: `UE4Editor-Cmd.exe <Project> -run=GenerateCallableCatalog`

## Catalog Refresh

Call `generate_catalog()` via MCP when:
- New plugins or modules have been added
- The catalog files are missing or outdated
- The user explicitly requests a refresh

Output goes to `.{agent}/callable_catalog/` by default.

## C++ Plugin Technical Details

**Module type:** `IModuleInterface` (not `UEditorSubsystem`) — avoids `EditorSubsystem` module dependency.

**TCP protocol:** Newline-delimited JSON on port 13090.
- Request: `{"command":"X","params":{...}}\n`
- Response: `{"success":true,...}\n`

**Thread model:** TCP listener thread accepts connections. Commands are dispatched to GameThread via `TPromise/TFuture` + `AsyncTask(ENamedThreads::GameThread)`, blocking the TCP thread until the result is ready. This ensures all UE API calls (reflection, Python execution) happen on the correct thread.

**Python soft dependency:** Uses `#if __has_include("IPythonScriptPlugin.h")` to compile with or without PythonScriptPlugin headers. At runtime, `IPythonScriptPlugin::Get()` returns null if the plugin is not loaded.

**Catalog generation:** Inline implementation (~200 lines) that scans `TObjectIterator<UClass>`, filters out internal classes (SKEL_, REINST_, DEPRECATED, etc.), iterates `TFieldIterator<UFunction>` for BlueprintCallable functions, and writes per-class JSON + category index.

## Tips

- **Static functions** can be called directly: `unreal.EditorAssetLibrary.load_asset(...)`
- **Instance functions** need an object reference first
- **Editor-only classes** (`editor_only: true`) only work in the editor, not packaged builds
- **Pure functions** (`is_pure: true`) have no side effects and are safe to call repeatedly
- Single expressions are auto-detected and evaluated for return values
- Multi-line scripts, imports, and control flow use file-execution mode
