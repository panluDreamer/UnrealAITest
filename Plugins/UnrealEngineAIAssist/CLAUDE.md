# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

UnrealEngineAIAssist is a UE4 editor plugin (C++ + Python) that bridges AI agents to Unreal Editor via a TCP/JSON protocol and the Model Context Protocol (MCP). It enables AI tools (Claude, Codebuddy, Cursor, Windsurf) to execute Python in the editor, introspect UHT reflection data, edit Blueprint graphs, and query a module knowledge graph.

```
AI Agent ──MCP (stdio)──> Python MCP Server ──TCP JSON (:13090)──> C++ Plugin ──UE API──> Editor
```

## Build & Setup

**Requirements**: UE 4.26+, Python 3.10+, MSVC/Clang

This is an Unreal Engine editor plugin. Build via standard UE workflow:
```bash
GenerateProjectFiles.bat   # or .sh on Mac/Linux
# Then build in Visual Studio / Rider / make
```

Verify plugin loaded: editor log should show `UnrealEngineAIAssist: Listening on 127.0.0.1:13090`

**Setup AI client integration:**
```bash
python setup.py                    # Auto-detect client
python setup.py --client claude    # Specific client
python setup.py --check            # Verify setup
python setup.py --uninstall        # Remove
```

Setup creates `{PluginDir}/.{client}/` with symlinks to Skills/ and Agents/, generates `.mcp.json`, installs Python deps (`mcp[cli]>=1.4.1`), and merges permissions into `settings.local.json`.

**No tests, linting, or CI are configured.**

## Architecture

### C++ Editor Module (`Source/UnrealEngineAIAssist/`)

Two source files:

- **UnrealEngineAIAssistModule.cpp/h** — Main module. Starts a TCP listener thread (`FBridgeRunnable`) on port 13090. Receives newline-delimited JSON commands, dispatches them to GameThread, and returns results. Commands: `exec_python`, `describe_object`, `generate_catalog`, `get_log`, `reflect` (raw property access bypassing UHT permission gates). Includes `FLogCapture` ring buffer (500 entries) for editor log capture. Conditionally links `PythonScriptPlugin` via `HAS_PYTHON_SCRIPT_PLUGIN` preprocessor define (auto-detected in Build.cs).

- **BlueprintEdGraphUtils.cpp/h** — `UBlueprintFunctionLibrary` exposing EdGraph pin/node operations as UFUNCTIONs. Wraps non-reflected APIs (MakeLinkTo, CreatePin, etc.) so AI agents can edit Blueprint graphs through `exec_python`. Key operations: graph discovery, node add/remove, pin connect/break/set-default, K2Node_CallFunction configuration, Blueprint compilation.

**Build.cs dependencies**: Core, CoreUObject, Engine, Networking, Sockets, Json, JsonUtilities (public); UnrealEd, BlueprintGraph, KismetCompiler (private). PythonScriptPlugin is a soft/optional dependency.

### C++ Runtime Module (`Source/UnrealEngineAIAssistRuntime/`)

- **DeviceBridgeModule.cpp/h** — TCP client that runs on-device (Android/iOS). Connects to a host-side `devbridge serve` instance, receives JSON commands, and executes them via `GEngine->Exec()`. Supports `exec_console`, `exec_unlua`, `get_cvar`, `set_cvar`, `get_log`, `get_info`. Trigger via `-AIAssistDeviceBridgeHost=<ip>:<port>` command-line arg or `AIAssistDeviceBridge <ip>:<port>` console command. DeveloperTool module type (excluded from Shipping builds).

### Python MCP Server (`Skills/ue-python-script/mcp_server/`)

~180-line FastMCP server (`unreal_agent_bridge_mcp.py`) that translates MCP tool calls into TCP JSON messages to the C++ plugin. Large responses are dumped to files under `.{client}/mcp_output/` to avoid filling AI context windows.

### Skills (`Skills/`)

Eight self-contained skills (no cross-skill dependencies), each with a `SKILL.md` activation script:

| Skill | Purpose |
|-------|---------|
| `ue-python-script` | Execute UE functions via MCP tools; houses the MCP server |
| `device-bridge` | Execute console commands, CVars, and Lua on connected Android devices |
| `renderdoc-capture` | Analyze RenderDoc GPU frame captures |
| `ue-knowledge-init` | Bootstrap module knowledge graph from Build.cs files |
| `ue-knowledge-update` | Incrementally update knowledge graph after code changes |
| `ue-knowledge-reader` | Query module relationships and architecture |
| `ue-agent-builder` | Create/update agents and skills dynamically |
| `sync-claude` | Initialize/sync `.claude/` directory (run after clone or adding skills) |

### Agents (`Agents/`)

Four example agents (ArtTechnician, DataDesigner, GameplayProgrammer, LevelDesigner) provided as templates. All share `Agents/RULE.md` which is loaded on activation. Use `ue-agent-builder` skill to create new domain-specific agents.

### Client Deployments (`.claude/`, `.codebuddy/`)

Per-client isolation via symlinks. Each contains `skills/`, `agents/`, `callable_catalog/`, `mcp_output/`, and `settings.local.json`. Generated by `setup.py`.

### Knowledge Graph (`Knowledge/`)

Shared knowledge data lives at the plugin root in `Knowledge/`, alongside `Agents/` and `Skills/`. Contains `module_graph.json`, `shader_map.json`, `modules/` summaries, etc. Generated by the `ue-knowledge-init` skill.

## Configuration

| Parameter | Default | Override |
|-----------|---------|----------|
| TCP Port | 13090 | `-AgentBridgePort=N` (editor arg) or `AGENT_BRIDGE_PORT` env |
| TCP Host | 127.0.0.1 | `AGENT_BRIDGE_HOST` env |
| Agent Dir | .claude | `AGENT_DIR_NAME` env |
| Plugin Dir | Auto-detected | `AGENT_BRIDGE_PLUGIN_DIR` env |

`plugin.config.json` is written automatically by the C++ plugin on editor startup with `engine_dir`, `plugin_dir`, and `engine_version`. The MCP server and `setup.py` read this file — no manual path configuration needed.

## Key Safety Rules (from Agents/RULE.md)

These apply when writing `exec_python` scripts targeting the editor:

1. **No PIE mutation** — never modify editor-world actor properties while Play-In-Editor is running
2. **No `ACTOR ADD`** — crashes the editor; use `BeginDeferredActorSpawnFromClass`
3. **Rotator keyword args** — always `unreal.Rotator(pitch=0, yaw=90, roll=0)` (positional order is roll, pitch, yaw)
4. **Deferred spawn** for Tick actors (PointLight, DirectionalLight, SkyLight, CameraActor) via `register_slate_post_tick_callback`
5. **Save before PIE** — save modified assets before entering Play-In-Editor
6. **One script, one task** — prefer single comprehensive scripts over multiple `exec_python` round-trips

## API Discovery Path

Before calling `exec_python`, always follow this progression:
```
L0: common-operations.md           ← covers 80% of tasks
L1: catalog_index.json              ← find the right class/category
L2: classes/{ClassName}.json        ← exact function signatures
L3: describe_object / reflect       ← live introspection (last resort)
```
Never jump to L3 without checking L0-L2 first. UE Python bindings differ significantly from C++ API.
